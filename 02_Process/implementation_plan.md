# 成本逼近法住宅范本对齐与费用测算实施计划

## 审阅结论

现阶段方案的总体方向正确：征收地类应与土地用途解耦，省级标准从正式附件读取，市县级配套政策允许校正，所有费用均由结构化参数和 `Decimal` 计算链生成。

原计划中“费用描述与 2025018 号地块定稿范本严格一字不差对齐”需要修正。2025018 号报告属于工业/林地场景，不能逐字复制到宝珍街住宅/水田场景。最终采用以下边界：

* 宝珍街住宅报告作为成本逼近法章节顺序、住宅参数说明、年期修正、区位修正和最终求价结构的主要范本。
* 2025018 号定稿报告用于补充征地补偿、税费政策依据及详细公式的写法。
* 县市、征收地类、用途、估价期日和确认后的外部测算结果均动态代入，禁止硬编码范例参数。

### 3. 宝珍街九步正文结构

运行母版中的成本逼近法旧固定块由后处理器物理替换，最终顺序固定为：

1. 土地取得费。
2. 相关税费及土地取得费、相关税费一览表。
3. 土地开发费及开发费用分项一览表。
4. 投资利息。
5. 投资利润。
6. 土地增值收益。
7. 土地使用年期修正系数。
8. 区位修正系数、修正规则表及修正结果表。
9. 成本逼近法价格计算。

前端第五部分和 Word 使用同一份动态正文、表格与计算结果。动态表格行禁止跨页拆分。

2. 表3-2 平均安置农业人口数确定表。
3. 表3-3 估价对象土地取得费及相关税费一览表。

### 4. 简化区位修正

住宅默认采用宝珍街范例的固定幅度等级修正，不再要求估价师使用 yaahp/AHP：

```text
因素修正率 = 等级档位 × 等级修正幅度
优/较优/一般/较劣/劣 = +2/+1/0/-1/-2 档
区位修正率 = 各因素修正率直接求和
```

估价师只需填写因素说明、选择优劣度并确认。高级加权修正模式继续保留，供确有需要的项目使用。

## LibreOffice 转换口径

本轮确认 `soffice.com` 超时不是因为文件过长。复测现象为：

* 同一小型 DOCX 直接从中文工程路径转换会卡住直到超时；
* 将 DOCX 复制到 `C:\tmp` 等纯英文临时目录后，同一 `soffice.com` 可正常生成 PDF；
* 动态收益法 DOCX 使用英文临时目录中转后也可成功转换。

因此正式报告导出和成交公告证据页生成均改为：

1. 将待转换 DOCX 复制到 ASCII 临时目录。
2. 将 LibreOffice `UserInstallation` profile 放在同一 ASCII 临时目录。
3. 让 `soffice.com` 只读写该临时目录。
4. 转换成功后复制 PDF 回原目标路径。

该策略不改变用户侧归档目录和证据快照目录，只隔离 LibreOffice 对中文路径、长文件名和残留 profile 锁的敏感性。
